# Contributing to the Geodynamics Education Modules

The Geodynamics Education Modules (GEMS) are a community project that lives by the participation of the geodynamics
community. That includes you! Our goal is to build an inclusive
and participatory community. We are happy that you are interested in
participating.

## Asking and answering questions about AxiSEM3D

For questions about the GEMs, please use the
[Education and Outreach](https://community.geodynamics.org/c/eo/36) forum.

Please note that everyone participating in the forum is volunteering
their time to help make our community a friendly and helpful
place. Therefore, please keep your inquiries
polite, take some effort to make them easily readable, and
include all useful information. If there is an issue or bug and it is 
something you can test on your own, please test it first,
before asking a question on the forum.

There is no guarantee that we can or will help with your problem, but
you are welcome to contact us. 

We look forward to your discussions, suggestions, and improvements.

## Reporting bugs

Help the community by reporting any bugs that you
may find. We keep track of all open issues related to the GEMs
[here](https://github.com/geodynamics/geodynamics_education_modules/issues).


## Making AxiSEM3D better

The GEMs are a community project. We encourage contributions of all kinds. 
Much appreciated contributions include new examples,
extending the documentation, and in particular fixing typos
or updating outdated documentation. 

If you consider making a
large contribution, please open a new
[issue](https://github.com/geodynamics/geodynamics_education_modules/issues/new) or 
[discussion](https://community.geodynamics.org/c/eo/36), to discuss
your idea with the community. This allows others who are interested or who have similar ideas to give you early
feedback and prevents conflicts.


### Getting started with git and GitHub

If you are new to using git or the GitHub platform, you may find it
helpful to review [video lecture
32.8](http://www.math.colostate.edu/~bangerth/videos.676.32.8.html), which
should be enough to help you get started with using GitHub or possibly
contributing to the GEMs itself. Alternatively, GitHub provides a helpful
guide on the process of contributing to an open-source project
[here](https://opensource.guide/how-to-contribute/).

### Opening pull requests

To make a change to the GEMs, you should:

- Create a
[fork](https://guides.github.com/activities/forking) (through GitHub) of
the code base.
- Create a separate
[branch](https://guides.github.com/introduction/flow/) (sometimes called a
feature branch) on which you do your modifications.
- You can propose that your branch be merged into the GEMs
code by opening a [pull request](https://guides.github.com/introduction/flow/).
This will give others a chance to review your code.

We follow the philosophy that no pull request (independent of the author) is
merged without a review from one other member of the community, and approval of
one of the maintainers. This applies to maintainers as well as to first-time
contributors. We know that a review can be a daunting process, but pledge to
keep all comments friendly and supportive, as you can see in the list of 
[open
pull requests](https://github.com/geodynamics/geodynamics_education_modules/pulls)! We are as
interested in making the GEMs better as you are!

While this seems very
formal, keeping all of the code review in one place makes it easier to
coordinate changes to the code (and there are usually several people making
changes to the code at once). This process is described at length in the
deal.II video [lecture
32.8](http://www.math.colostate.edu/~bangerth/videos.676.32.8.html).  Please do
not hesitate to ask questions about the workflow on the forum if you are
not sure what to do.

### Coding conventions


We use [PEP8](https://peps.python.org/pep-0008/) standard Python coding conventions to keep the coding style consistent.

### Changelog entries
If your new pull request creates a change that is noticeable to AxiSEM3D users,
please add an entry to our
[changelog](https://github.com/geodynamics/geodynamics_education_modules/blob/main/CHANGELOG.md). 
Start your newline with an action verb such as added, implemented, changed, fixed, removed, etc., explain the nature of your change,
include your name and date in the file as shown (this ensures you will get credit for your work), and lastly, include the pull request number.

## Acknowledgment of contributions

This project is grateful for every contribution! 
Your contribution is *formally* acknowledged in several ways:

- Every commit that is merged into the GEM repository makes you part of
  the growing group of GEM
  [contributors](https://github.com/geodynamics/geodynamics_education_modules/graphs/contributors?from=3%2F14%2F2026).
- For every release, the most significant entries of our
  [changelog](https://github.com/geodynamics/geodynamics_education_modules/blob/main/CHANGELOG.md)
  are selected to generate our release announcements. Additionally, all entries
  remain available for all previous releases of the GEMs inside this file.
- If you contribute a significant example or section to the documentation, you will be acknowledged in [Authors.md](https://github.com/geodynamics/geodynamics_education_modules/blob/main/AUTHORS).
- This project receives oversight from the CIG [Education Working Group](https://geodynamics.org/groups/education) which meets to discuss the direction
  of the project and to consider inviting new members to join the group of
  Maintainers and Authors.
  New members are typically invited based on the following criteria:
  - A demonstrated commitment to advancing the project's goals and supporting
    the GEMs community;
  - Meaningful contributions to the GEMs. These contributions may of course be
    (to )a notebook, but can also include documentation or active engagement on the forum;
  - Consistent and active involvement in the project for over a year,
    beyond participation in meetings;
  - A collaborative mindset and a constructive, solution-oriented approach
    to discussions.

  The group of Authors who have contributed to developing the philosophy, direction, and or code to the 
  project are listed in the
  [AUTHORS.md](https://github.com/geodynamics/geodynamics_education_modules/blob/main/AUTHORS)
  file in the main repository.
